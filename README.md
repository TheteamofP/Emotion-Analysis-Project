## 2、选题要求

<br>

### 2.1、选题系统具体描述
|  系统名称     |  EmotionAnalyzer - 情感分析器       | 
| :----: | :----: |
|  系统功能1    | 从社交媒体、电子商务平台评论、新闻文章或客户反馈中提取文本 |
|  系统功能2    |  对提取的文本进行预处理（如去噪、分词、去停用词等）|
|  系统功能3    |  通过情感分类模型对处理过后的文本的情感倾向进行分类 |
|  系统功能4    |  将情感分析分类结果（正面、负面或中立）展示给用户 |
|  系统功能5(附加功能，看条件实现)    |  提供可视化工具（如曲线，表格和词云等）分析情感趋势、情感波动和重点话题 |

<br>

### 2.2、预期用户量
+ ##### 由于目标用户群体是公司相关的分析决策员工，系统上线后预期用户量在 500-1000 人之间。随着推广和使用的逐步深入，用户量将可能上升到 2000-3000 人。

## 3、团队贡献分评估方法

<br>

### 3.1、分配原则
+ **公平性**
  + 确保每个成员的贡献得到公正的评估和相应的回报。
+ **透明度**
  + 分配规则和过程对所有团队成员公开透明。
+ **激励性**
  + 鼓励成员积极参与项目，提高工作效率和质量。
+ **灵活性**
  + 允许根据项目实际情况和成员表现进行适当调整。

<br>

### 3.2、分配规则
|  工作量（40%）    |  质量（30%）    |  团队合作（20%）    | 创新和解决问题能力（10%）|
| :----: | :----: | :----: |:----: |
|  总和为32分    |  总和为24分    | 总和为16分     | 总和为8分 |
|  根据成员完成的工作量和任务难度进行分配。    | 根据成员工作成果的质量，包括代码质量、文档完整性等。     |  根据成员对团队合作的贡献，如帮助同伴、共享知识等。    |根据成员提出的创新想法和有效解决问题。|

<br>

### 3.3、分配流程
+ **（1）自我评估**
  + 每个成员根据自己的表现对四个分配方面进行自我分数评定，并说明评定根据。
+ **（2）队员互评**
  + 团队成员相互评估其他队员四个分配方面得分数，并说明评定根据，以确保评估的客观性。
+ **（3）项目经理审核**
  + 项目经理根据自我评估和队员互评的结果进行审核，并提出初步分配方案。
+ **（4）团队讨论**
  + 团队成员就初步分配方案进行讨论，确保分配结果都有根有据。
+ **（5）最终确定**
  + 项目经理根据讨论结果最终确定每个成员的团队贡献分。

<br>

### 3.4、记录与反馈
+ **记录**
  + 所有工作与团队贡献分分配过程和结果都将被记录，以备后续参考，体现工作留痕与表现的重要性。
+ **反馈**
  + 项目进行中后，团队成员可以根据工作对分配规则提出反馈与改进，团队讨论决定是否增删规定，以便在项目中每个人的贡献能够有该有的分数收获。
